================================================================================
HOW THE SCRIPTS WORK
================================================================================

VERIFICATION FLOW:
  verify-integrations.sh
    └─ Checks .env.local for all required variables
    └─ Validates variable formats (Stripe keys, JWT tokens, etc)
    └─ Tests connectivity to external services
    └─ Reports: PASS, WARN, or FAIL for each check
    └─ Exit: 0 if all critical checks pass, 1 otherwise

DEPLOYMENT FLOW:
  deploy-to-vercel.sh
    ├─ Prerequisites check (Vercel CLI, login, .env.local)
    ├─ Load environment variables from .env.local
    ├─ Link project to Vercel
    ├─ Configure environment variables in Vercel
    ├─ Set up build settings (vercel.json)
    ├─ Initiate deployment to preview or production
    └─ Provide next steps for domain configuration

TESTING FLOW:
  test-endpoints.sh
    ├─ Verify server is running
    ├─ Test each API endpoint with sample requests
    ├─ Verify response codes (200, 400, 404)
    ├─ Check error handling
    ├─ Report pass/fail for each test
    └─ Exit: 0 if all tests pass, 1 otherwise

QUICK START COMMANDS

   $ npm run landing:dev &
   $ ./test-endpoints.sh
   $ kill %1

   $ ./test-endpoints.sh https://inkluyo.vercel.app

   $ ./deploy-to-vercel.sh production

6. Configure domain:
   (Follow DEPLOYMENT_GUIDE.md section "Configure Spaceship Domain")

7. Test production:
   $ ./test-endpoints.sh https://inkluyo.spaceship.com
